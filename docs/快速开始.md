# NeoFeed 快速开始指南

## 🎯 30 秒启动

### 方式 1: 一键启动（推荐）

```bash
chmod +x start.sh
./start.sh
```

### 方式 2: 分步启动

**终端 1 - 后端**:
```bash
cd /Users/Zhuanz/NeoFeed
python -m uvicorn api.main:app --reload
```

**终端 2 - 前端**:
```bash
cd /Users/Zhuanz/NeoFeed/web
npm run dev
```

## 📋 首次使用

### 1. 安装依赖

```bash
# Python 依赖
pip install -r requirements.txt

# 前端依赖
cd web && npm install
```

### 2. 初始化数据库

```bash
cd database
python init_db.py
```

### 3. 配置环境（可选）

```bash
cp .env.example .env
# 编辑 .env，添加 OPENAI_API_KEY
```

## 🎮 基本使用

### 保存文本

1. 打开 `http://localhost:3000`
2. 在输入框粘贴文本
3. 点击「保存」

### 保存链接

1. 复制任意网页 URL
2. 粘贴到输入框
3. 系统自动抓取内容
4. 点击「保存」

### 启用 AI 处理

1. 勾选「启用 AI 处理」
2. 保存内容
3. 等待几秒，AI 自动生成：
   - 📝 3 句话摘要
   - 🏷️ 分类
   - 🔑 关键词

## 🔧 常见问题

### Q: 前端无法连接后端？

**A**: 检查后端是否启动：
```bash
curl http://localhost:8000/health
```

### Q: AI 处理不工作？

**A**: 两种可能：
1. 没有配置 `OPENAI_API_KEY`（正常，AI 可选）
2. 在 `.env` 中设置 `ENABLE_AI_PROCESSING=true`

### Q: 无法抓取 URL？

**A**: 检查：
1. URL 格式是否正确
2. 网络连接是否正常
3. Jina Reader API 是否可用

### Q: 数据保存在哪里？

**A**: 本地 SQLite 数据库：`database/neofeed.db`

## 🌐 局域网部署

让同一 WiFi 下的手机、平板或其他电脑通过浏览器访问本机运行的前端：

### 1. 用 LAN 模式启动前端

```bash
cd web
npm run dev:lan
```

会监听 `0.0.0.0:3000`，终端里会显示 **Network: http://0.0.0.0:3000**。

### 2. 允许本机 LAN IP 访问

若看到 Next.js 提示 `Cross origin request detected from x.x.x.x`，需在 `web/next.config.ts` 的 `allowedDevOrigins` 里加入**本机局域网 IP**（如 `10.23.196.167` 或 `192.168.1.100`）。  
查看本机 IP：`ifconfig`（Mac/Linux）或 `ipconfig`（Windows）。

### 3. 其他设备访问

在同一局域网内的设备浏览器中打开：`http://<你的电脑IP>:3000`（例如 `http://10.23.196.167:3000`）。

### 4. 登录与回调（若使用 Supabase 等）

如需在局域网下登录、回调正常，在 `web/.env.local` 中设置：

```bash
NEXT_PUBLIC_APP_URL=http://<你的电脑LAN IP>:3000
```

这样邮件里的登录链接、OAuth 回调等会指向该地址。

### 5. 移动端打不开怎么办？

- **同一 WiFi**：手机必须和电脑连在**同一个 WiFi**，不要用手机蜂窝网络。
- **本机防火墙**（最常见）：
  - **Mac**：系统设置 → 网络 → 防火墙 → 选项 → 允许「Node」或「node」传入连接；或开发时暂时关闭防火墙测试。
  - **Windows**：Windows 安全中心 → 防火墙 → 允许应用通过防火墙 → 勾选 Node 或对应开发工具。
- **地址要写全**：在手机浏览器地址栏输入 `http://10.23.199.165:3000`（把 IP 换成你电脑的 LAN IP），不要漏掉 `http://` 和 `:3000`。
- **先试 /landing**：若根路径打不开，可试 `http://<电脑IP>:3000/landing`。
- **确认电脑已开 dev:lan**：终端里要有 `Network: http://0.0.0.0:3000` 且无报错。
