# NeoFeed 页面结构说明

## 📐 整体布局

```
┌─────────────────────────────────────────────────┐
│  左侧导航栏 (固定)    │    主内容区域         │
│  256px                │    自适应             │
│                       │                        │
│  NeoFeed Logo         │                        │
│  ─────────────        │                        │
│  👤 个人信息          │     根据路由          │
│  📚 历史记录          │     显示不同页面       │
│  📖 待阅读信息        │                        │
│                       │                        │
│  ─────────────        │                        │
│  版本 v0.2.0          │                        │
└─────────────────────────────────────────────────┘
```

---

## 📄 页面路由

### 1. 首页 `/`
**功能**：保存新内容

**布局**：
- 中间居中显示输入表单
- 包含：
  - 标题输入框（可选）
  - 内容输入框（大文本框）
  - AI 处理开关
  - 保存按钮

**特点**：
- 极简设计，专注输入
- 支持文本和 URL
- 自动提示功能

---

### 2. 个人信息 `/profile`
**功能**：查看统计和账户信息

**内容**：
1. **统计卡片**（4个）
   - 📊 总条目
   - ✅ 已处理
   - ⏳ 待处理
   - ❌ 失败

2. **账户信息**
   - 用户头像
   - 邮箱
   - 使用说明

3. **功能设置**
   - AI 自动处理
   - 网页抓取
   - 数据导出

---

### 3. 历史记录 `/history`
**功能**：查看所有已保存的信息

**内容**：
- 顶部统计（总记录数 + 刷新按钮）
- 信息卡片列表：
  - 标题 + URL（如果有）
  - AI 摘要（如果有）
  - 分类标签
  - 关键词
  - 内容预览（可展开/收起）
  - 状态标签
  - 保存时间

**特点**：
- 按时间倒序排列
- 支持展开全文
- 显示处理状态

---

### 4. 待阅读信息 `/reading`
**功能**：查看未处理的内容

**内容**：
- 顶部统计（待阅读数量 + 刷新按钮）
- 卡片式布局（2列网格）：
  - 标题
  - URL 链接
  - 内容摘要
  - AI 摘要（如果有）
  - 保存时间
  - "待阅读" 标签

**筛选条件**：
- 只显示 `status = 'pending'` 的条目

**特点**：
- 卡片式展示，适合快速浏览
- 点击可跳转到原链接

---

## 🎨 设计特点

### 左侧导航栏
- **固定宽度**：256px
- **固定位置**：左侧，高度占满屏幕
- **导航高亮**：当前页面路径高亮显示
- **底部信息**：显示版本号

### 主内容区域
- **左边距**：256px（为导航栏留空间）
- **背景渐变**：蓝紫色渐变
- **最大宽度**：各页面最大 1280px，居中显示
- **内边距**：32px

### 配色方案
- **主色**：蓝色 `#2563eb` 到紫色 `#9333ea` 的渐变
- **背景**：白色 + 淡蓝紫渐变
- **文字**：
  - 主标题：`#1f2937`（深灰）
  - 正文：`#4b5563`（中灰）
  - 辅助：`#6b7280`（浅灰）
- **状态色**：
  - 成功/已处理：绿色 `#10b981`
  - 待处理：蓝色 `#3b82f6`
  - 失败：红色 `#ef4444`
  - 待阅读：橙色 `#f59e0b`

## 🔄 交互流程

### 保存内容流程
1. 在首页输入内容
2. 点击「保存」
3. 后端处理
4. 显示成功提示
5. 可在「历史记录」或「待阅读信息」查看

### 查看内容流程
1. 点击左侧导航
2. 进入对应页面
3. 浏览卡片列表
4. 点击展开查看全文

## 📱 响应式设计

### 桌面端（>1024px）
- 左侧导航固定 256px
- 主内容区域自适应
- 历史记录：单列
- 待阅读：2列网格

### 平板端（768px - 1024px）
- 保持桌面布局
- 待阅读：2列网格
