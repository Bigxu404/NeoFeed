# NeoFeed 系统深度演进报告 (2026-01-15)

## 1. 核心设计变革：洞察中心 (Insight Center) 重塑

本次更新对“洞察中心”进行了结构性的重构，从原本的单一流展示进化为**“双轴并行”**的控制台布局。

### 1.1 两栏式智能控制台
- **控制塔 (Control Tower - 左侧 380px)**：
    - **视觉风格**：采用经典的 Fallout (辐射) 废土终端风格，结合 CRT 扫描线与闪烁效果，营造沉浸式情报分析感。
    - **间隙优化**：通过 `pl-12` 的大幅左侧留白与面板悬浮设计，解决了界面贴边的压迫感。
    - **功能集成**：集成了 RSS 信号源管理与“周报协议”配置。
- **情报流 (Intelligence Stream - 右侧)**：
    - **视觉风格**：采用极简的“知识星球”风格，强调文字可读性，去除了冗余的绿色渲染。
    - **交互逻辑**：鼠标悬浮展示点击交互响应，点击后以模态弹窗形式展开深度阅读，支持 AI 总结与原网页跳转。
    - **布局补偿**：通过缩小控制塔宽度与增加中间渐变边界线，为阅读区域提供了更广阔的视觉空间。

### 1.2 RSS 智能引擎：全量拦截与结构化总结
- **全量拦截机制**：取消了 Top 7 筛选逻辑，单次同步全量接收最多 20 条最新动态。
- **同步即清空逻辑**：引入了“信号刷新即覆盖”策略，每次抓取新内容时会自动清空旧数据，确保情报流永远保持极高的新鲜度。
- **科研级四段式总结**：AI 针对每一条情报生成结构化分析（研究主题、研究方式、研究结果、一句话闭环总结），大幅降低深度文章的阅读门槛。
- **可读性专项优化**：大幅提升了 AI 中文总结的字号，展示一句话总结作为卡片核心，去除了强制大写转换。

## 2. 周报系统 (Weekly Protocol) 颗粒度拆解

周报生成逻辑从“单一任务”进化为“双轨定制化”方案。

### 2.1 任务分离 (Segregation of Duties)
- **洞察周报 (Insight Report)**：仅抓取用户手动捕获的 `feeds` 数据。
- **订阅周报 (RSS Report)**：仅汇总来自 RSS 订阅源的 `discovery_stream` 内容。

### 2.2 深度定制化能力
- **独立提示词 (Prompts)**：为两种周报分别提供独立的 AI 指令编辑框。
    - *RSS 专用提示词*：特别强化了 AI 分类汇总能力，限制输出 20 条高价值信息，并强制包含 URL 溯源。
- **精准调度配置**：支持按周（周一至周日多选）和具体时间（精确到分钟）为每种报告设置独立的发送频率。

## 3. 系统架构与底层优化

### 3.1 神经核心 (Neural Core) 回归
- 在系统设置中完整恢复了 LLM API 的配置面板。
- 统一了全站导航栏 (`DashboardHeader`)，确保工作台、星系、洞察、设置页面的操作路径与视觉位置完全对齐。

### 3.2 任务引擎 (Inngest) 健壮性增强
- **环境自适应**：修复了本地开发环境下 Inngest 的 401 Event Key 报错，实现了本地 Dev Server 与生产环境的无缝切换。
- **缺失组件找回**：通过 Git 恢复了被误删的核心组件库（`web/hooks`, `web/components/dashboard` 等），彻底解决了性能卡顿与构建报错问题。
- **依赖库补全**：安装并集成了 `linkedom` (本地抓取回退方案) 和 `rss-parser` (RSS 协议解析)。

## 4. UI/UX 体验细化
- **去线框化设计**：移除了面板的硬性边框，通过阴影与背景模糊实现自然分区，使界面更显现代感。
- **通知系统**：集成了 `sonner` 全局 Toast 提示，为信号同步、周报触发等异步操作提供即时反馈。
- **清除彩蛋**：移除了“Smith”等干扰性的 Easter Egg，回归纯粹的生产力工具属性。
